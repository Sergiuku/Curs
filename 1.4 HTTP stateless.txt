HTTP este un protocol stateless. Numele vine de la Hyper Text Transfer Protocol, acesta fiind un protocol pentru transmiterea informatiilor din WWW (WORLD WIDE WEB).
Este numit stateless deoarece fiecare comanda solicitata este executata independent, fara a se tine cont de cererile executate inainte, tratand fiecare cerere pentru o pagina web ca o conexiune unica.
Se mai poate spune ca la fiecare conectiune dintre client si server toate informatiile se pierd la momentul cand tranzactia se termina.
Un protocol stateless nu necesita strangerea de informatii despre un user pe durata comunicarii intre server si client.
Datorita acestui protocol comunicarea dintre client si server se face cu minimul de cantitate de date care trebuie transferata.
In cazul unei aplicatii web exista o nevoie de monitorizare si stocare a informatiilor useului ceea ce un protocol stateless nu poate face, dar exista solutii cum ar fi cookie-urile, sesiunile sau chiar variabilele.
Pentru a comunica prin acest protocol exista anumite metode de solicitare. 
Aceste metode sunt:
1.GET folosita pentru a obtine date de la server.
2.HEAD este folosita atunci cand userul doreste sa afle informatii despre head-ul documentului, cum ar fi charset-ul, lungimea continutului,etc.
3.Metoda DELETE se foloseste pentru a sterge date.
